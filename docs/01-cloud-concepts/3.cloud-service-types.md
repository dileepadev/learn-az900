## Infrastructure as a Service (IaaS)

- Infrastructure as a Service (IaaS) provides virtualized computing resources over the internet.
- Customers can rent virtual machines, storage, and networking resources on a pay-as-you-go basis.
- IaaS allows organizations to avoid the cost and complexity of purchasing and managing physical servers and data centers.
- Focus on managing applications, data, runtime, middleware, and the operating system while the cloud provider manages the underlying infrastructure.

## Platform as a Service (PaaS)

- Platform as a Service (PaaS) provides a platform that allows customers to develop, run, and manage applications without worrying about the underlying infrastructure.
- It includes tools and services for application development, testing, and deployment.
- PaaS enables developers to focus on building applications rather than managing infrastructure.
- Focus on managing applications and data while the cloud provider manages the underlying infrastructure and platform.

## Software as a Service (SaaS)

- Software as a Service (SaaS) provides access to software applications over the internet.
- Customers can use the software without having to install or manage it on their own devices.
- Examples include email services, customer relationship management (CRM) software, and collaboration tools.
- SaaS allows users to access applications from any device with an internet connection, providing flexibility and ease of use.
- Focus on using the software and managing data within the application while the cloud provider manages everything else, including the application itself.

## Shared Responsibility Model

- In cloud computing, the shared responsibility model defines the division of responsibilities between the cloud service provider and the customer.
- Some responsibilities always remain with the customer, regardless of the service type. These include information and data, devices such as mobile phones and PCs, and accounts and identities.
- Some responsibilities vary depending on whether the service is SaaS, PaaS, or IaaS. These include identity and directory infrastructure, applications, network controls, and the operating system.
- Some responsibilities transfer to the cloud provider. These include physical hosts, the physical network, and the physical datacenter.
- For **IaaS**, the provider is responsible for the physical infrastructure, while the customer is responsible for the operating system, applications, and data.
- For **PaaS**, the provider manages the infrastructure and platform, while the customer is responsible for the applications and data.
- For **SaaS**, the provider manages everything, including the application, while the customer is responsible for using the software and managing their data within the application.
- In an on-premises environment, the customer is responsible for the entire stack, from the physical datacenter up to the data and user access.

| Responsibility | SaaS | PaaS | IaaS | On-premises |
| --- | --- | --- | --- | --- |
| Information and data | Customer | Customer | Customer | Customer |
| Devices (mobile and PCs) | Customer | Customer | Customer | Customer |
| Accounts and identities | Customer | Customer | Customer | Customer |
| Identity and directory infrastructure | Shared | Shared | Customer | Customer |
| Applications | Provider | Shared | Customer | Customer |
| Network controls | Provider | Shared | Customer | Customer |
| Operating system | Provider | Provider | Customer | Customer |
| Physical hosts | Provider | Provider | Provider | Customer |
| Physical network | Provider | Provider | Provider | Customer |
| Physical datacenter | Provider | Provider | Provider | Customer |

- Understanding the shared responsibility model is crucial for ensuring security and compliance in the cloud.
- Customers must be aware of their responsibilities and take appropriate measures to protect their data and applications in the cloud.
